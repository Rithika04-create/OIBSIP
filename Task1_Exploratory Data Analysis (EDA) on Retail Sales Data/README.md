🌟 Oasis Infobyte Internship Project

📊 Exploratory Data Analysis (EDA) on Retail Sales Data

🚀 Project Overview

"Raw data doesn't tell stories — but EDA does."

This project is a complete Exploratory Data Analysis on a real-world retail sales dataset, performed during my Data Analytics Internship at Oasis Infobyte. The goal was to go beyond basic statistics and uncover hidden patterns, customer behavior, and actionable business insights using Python.

🎯 Objectives

✅ Clean and preprocess messy retail data
✅ Handle missing values & outliers effectively
✅ Identify peak sales periods and underperforming categories
✅ Uncover seasonality and customer purchasing trends
✅ Generate business recommendations from data

🛠️ Tech Stack

Tool	                           Purpose

Python	                     Core programming
Pandas	                     Data manipulation
NumPy	                       Numerical operations
Matplotlib & Seaborn	       Data visualization
Google Colab	               Interactive environment


📁 Dataset Description

Source: Retail sales transaction data
Features include: Transaction date, product category, quantity sold, price, customer segment, store location, etc.
Time period: Multi-month sales records

🔍 Key Steps Performed

🧹 Data Cleaning

Handled missing values (dropped / imputed)
Removed duplicates
Fixed data types

📊 Exploratory Analysis

Statistical summaries
Trend analysis over time
Category-wise performance

📈 Visualization

Sales trends (line plots)
Category contribution (bar charts)
Correlation heatmaps
Outlier detection (boxplots)

💡 Insight Generation

Peak sales periods identified
Top performing product categories
Customer buying patterns by segment

💡 Key Business Insights

✅ Peak sales observed during weekends and end of month
✅ Electronics & Clothing are top-performing categories
✅ Discounts > 20% significantly boost volume but not profit
✅ Certain customer segments prefer morning vs evening purchases
🔥 Surprise finding: Highest sales occurred on Tuesdays, not weekends — challenging common retail assumptions
